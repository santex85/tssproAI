# Версионирование (SemVer + Git Tags)

Проект использует **семантическое версионирование (SemVer)** в связке с **Git Tags**. В проде крутятся только протегированные сборки; образы без тега (`latest`) не используются.

## Формат версии

`vMAJOR.MINOR.PATCH-MODIFIER`

- **MAJOR** — несовместимые изменения API.
- **MINOR** — новая функциональность с обратной совместимостью.
- **PATCH** — исправления без изменения API.
- **MODIFIER** — пререлиз: `alpha`, `beta`, `rc` и т.п.

Примеры:
- Альфа: `v0.1.0-alpha.1`, `v0.1.0-alpha.2`
- Бета: `v0.2.0-beta.1`
- Релиз-кандидат: `v1.0.0-rc.1`
- Стабильный релиз: `v1.0.0`

## Где хранится версия

- **Корень репозитория:** файл `VERSION` (одна строка, без префикса `v`) — единый источник истины для альфы/ручных релизов.
- **Фронт:** `frontend/package.json` → `version`, `frontend/app.json` → `expo.version`.
- **Бэкенд:** `backend/pyproject.toml` → `version`, `backend/app/main.py` → FastAPI `version` (OpenAPI).
- При появлении CI/CD версия для сборки берётся из **Git Tag** (например `git describe --tags --always`), а не из `VERSION` в артефактах.

## Процесс выпуска альфы (вручную)

1. Сделать нужные коммиты, код готов к альфе.
2. При необходимости обновить версию в `VERSION`, `package.json`, `app.json`, `pyproject.toml`, `main.py` (например `0.1.0-alpha.2`).
3. Закоммитить изменения версии.
4. Поставить аннотированный тег:
   ```bash
   git tag -a v0.1.0-alpha.1 -m "Release Alpha 1: базовый функционал tssAI"
   ```
5. Отправить тег на сервер:
   ```bash
   git push origin v0.1.0-alpha.1
   ```
6. Собрать и задеплоить образы **с этим тегом** (см. ниже). В проде использовать только образы с тегом версии.

## Сборка образов с тегом версии

Правило: **в проде крутятся только протегированные сборки, никакого `latest`.**

Локально (после создания тега) используйте цель Makefile:

```bash
make build-prod-tagged
```

Версия берётся из `git describe --tags --always` (например `v0.1.0-alpha.1` или короткий хеш). Образы помечаются как `st2-backend:$(VERSION)`, `st2-frontend:$(VERSION)`. Переопределить версию: `make build-prod-tagged VERSION=v0.1.0-alpha.2`.

В CI/CD пайплайн при пуше тега (например `v0.1.0-alpha.1`) должен:
1. Собрать backend и frontend.
2. Присвоить образам тег версии: `backend:v0.1.0-alpha.1`, `frontend:v0.1.0-alpha.1`.
3. Пушить образы в registry с этим тегом.
4. Деплой на прод — только по тегу образа (никакого `latest`).
